6. Arithmetic triple OT 生成提示词

对每个 triple，需要满足：

b = b0 + b1 mod 2^32
c = c0 + c1 mod 2^32 = a*b mod 2^32
展开：
c = (a0+a1)(b0+b1)
  = a0b0 + a0b1 + a1b0 + a1b1
其中 a0b0 和 a1b1 可以本地算，交叉项 a0b1 和 a1b0 需要通过 OT Extension 生成 additive shares。
要求：

4. 使用 ot_extension_ring.py；

6. 不要重构完整 a、b、c 给某一方。


7. A2B、B2A、安全比较和 MSB 提示词
请你帮我实现 A2B，也就是 arithmetic share 到 Boolean bit share 的转换。
背景：
Sonic 中 secure MSB 需要 bit 级表示，但输入 x 一开始是 arithmetic share。
两方分别持有 x0 和 x1，需要得到 x = x0 + x1 mod 2^32 的每一位 Boolean share。
要求：
1. 输入 xi 是当前方持有的 arithmetic share；
2. 输出 bits 列表，bits[i] 是第 i 位 Boolean share；
3. 用逐位 full-adder 思路处理 x0+x1；
4. carry 需要用安全 bit_and 计算；
5. 支持 numpy 数组；
6. bit_len 默认 32；
7. 不要重构 x。

请你帮我实现 B2A，也就是 Boolean share 到 Arithmetic share 的转换。
背景：
SReLU 中比较结果是 Boolean share，但后续要和 arithmetic share 的 x 相乘，所以需要转成 arithmetic share。
论文公式：
如果 x = x0 XOR x1，则
x = x0 + x1 - 2*x0*x1
要求：
1. 输入 xb_i 是当前方持有的 Boolean share；
2. 输出 arithmetic share；
3. 乘积 x0*x1 要调用 secure_mul；
4. 所有计算在 Z_(2^32) 环上完成；
5. 不要直接重构 Boolean 值。

请你帮我实现 secure_msb 和 secure_compare_zero。
背景：
Sonic 中 ReLU(x) = NOT(MSB(x)) * x，因此需要安全提取 x 的最高位。
x 是 arithmetic share，负数采用二补码表示。
要求：
1. secure_msb(xi) 输出 x0+x1 mod 2^32 的 MSB Boolean share；
2. 使用逐位 full-adder 计算 sum 和 carry；
3. carry 计算中的 AND 要调用 bit_and；
4. secure_compare_zero(xi) 返回 Boolean share：
   1 表示 x >= 0，0 表示 x < 0；
5. Boolean share 下取反只让一方 XOR 1；
6. 当前先实现功能级版本，后续再考虑论文中的 PPA 优化。


8. SReLU 代码提示词
请你帮我实现 Sonic 中的 SReLU。
论文原理：
ReLU(x) = max(0,x)
在二补码 fixed-point 表示下：
ReLU(x) = NOT(MSB(x)) * x
实现要求：
1. 输入 xi 是 arithmetic share；
2. 调用 secure_compare_zero 得到 gate 的 Boolean share；
3. 调用 b2a_secure 把 gate 转成 arithmetic share；
4. 调用 secure_mul 计算 gate * x；
5. 输出仍然是 arithmetic share；
6. 不要重构 x 或 gate；
7. 加入 profiler 统计 relu_calls 和 relu_time。


9. Secure truncation 代码提示词
请你帮我实现 fixed-point 推理中的 secure_trunc。
背景：
输入和权重都是 scale=2^f 时，乘法后结果变成 scale=2^(2f)，需要右移 f 位恢复到 2^f。
但是 x 是 arithmetic share，不能直接重构后右移。
同时负数采用二补码表示，不能简单做无符号右移，而要考虑符号位。
实现思路：
1. 先通过 A2B 把 arithmetic share 转成 bit shares；
2. 取第 31 位作为 sign bit；
3. 做 signed arithmetic right shift；
4. 高位用 sign bit 补齐；
5. 每一位通过 B2A 转回 arithmetic share；
6. 按 bit 权重 2^j 重组；
7. 输出截断后的 arithmetic share。
要求：
1. 不要重构明文；
2. 支持 numpy 数组；
3. 统计 trunc_calls 和 trunc_time；
4. 所有计算保持在 Z_(2^32) 环上。

11. SBN 代码提示词
请你帮我实现 Sonic 中的 Secure Batch Normalization。
论文中推理阶段 BN 可以折叠为：
z = eps1 * x + eps2
其中：
eps1 = gamma / sqrt(var + eps)
eps2 = beta - gamma * mean / sqrt(var + eps)
实现要求：
1. mpc/sbn.py 中实现 sbn_fixed；
2. 输入 x_i 是 arithmetic share；
3. eps1_i 和 eps2_i 也是 arithmetic share；
4. 支持 Conv 输出 [N,C,H,W] 和 FC 输出 [N,D]；
5. 如果 eps1_i shape 是 [C]，要 broadcast 到 [N,C,H,W]；
6. 如果 eps1_i shape 是 [D]，要 broadcast 到 [N,D]；
7. eps1*x 使用 secure_mul；
8. eps2 按 2^(2f) 编码后直接相加；
9. 最后用 secure_trunc 恢复 scale；
10. 不要重构中间结果。

请你帮我在 sonic_pytorch_models.py 中实现从 PyTorch BatchNorm 导出 Sonic SBN 参数的函数。
要求：
1. 输入 PyTorch BatchNorm 层；
2. 读取 weight(gamma)、bias(beta)、running_mean、running_var、eps；
3. 计算 eps1 = gamma / sqrt(running_var + eps)；
4. 计算 eps2 = beta - gamma * running_mean / sqrt(running_var + eps)；
5. 返回 numpy 数组，供后续 fixed-point 编码。


12. Secure MaxPool 和优化顺序提示词
请你帮我实现 Sonic 中的 Secure Max Pooling。
思路：
两个 secret shared 数的最大值可以写成：
max(x,y) = y + gate*(x-y)
其中 gate = 1 if x >= y else 0
实现要求：
1. 实现 secure_max2(x_i, y_i)；
2. 先计算 diff = x - y；
3. 用 secure_compare_zero(diff) 得到 gate 的 Boolean share；
4. 用 b2a_secure 转成 arithmetic share；
5. 用 secure_mul 计算 gate*diff；
6. 输出 y + gate*diff；
7. 实现 maxpool2d_secret，遍历池化窗口，多次调用 secure_max2；
8. 支持 NCHW 格式；
9. 当前先支持 padding=0。

Sonic 论文中提到 ReLU 和 MaxPool 可以交换顺序，用 MaxPool → ReLU 替代 ReLU → MaxPool，从而减少安全 ReLU 调用次数。
请你帮我实现一个优化版 CNN 推理函数 secure_fixed_cnn_sbn_pool_relu，要求流程为：
Conv2D
→ SBN
→ Secure MaxPool2D
→ SReLU
→ Flatten
→ Secret Linear
→ Secure Truncation
说明：
1. 先 MaxPool 再 SReLU；
2. 体现 Sonic 中减少 SReLU 调用数量的优化；
3. 所有输入和参数都是 arithmetic share；
